COMO PROBAR (Windows / PowerShell)
==================================

Objetivo
--------
Levantar backend + frontend y probar la sala compartida donde puedes elegir:
1) Space Invaders
2) Prueba WebSocket

Importante
----------
Este backend de pruebas NO necesita base de datos.
Ya esta configurado para arrancar sin DataSource/JPA.


LANZAMIENTO RAPIDO (COPY/PASTE)
-------------------------------
Usa exactamente estos comandos en este orden.

TERMINAL A (backend):
cd .\juegos1000tres-backend
.\mvnw.cmd spring-boot:run

TERMINAL B (frontend):
cd .\juegos1000tres-frontend
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python .\main.py

Luego abre la URL que imprime la consola del frontend:
- URL: http://localhost:<PUERTO_FRONTEND>
- Sala: http://localhost:<PUERTO_FRONTEND>/pruebas/sala-mentira

Si algo falla por puertos ocupados, ejecuta primero el paso 0 y repite.


0) Limpiar puertos si hubo ejecuciones previas
----------------------------------------------
Ejecuta esto en PowerShell (desde cualquier carpeta):

function Stop-PortProcess($port) {
	$connections = Get-NetTCPConnection -LocalPort $port -State Listen -ErrorAction SilentlyContinue
	if ($null -ne $connections) {
		$connections |
			Select-Object -ExpandProperty OwningProcess -Unique |
			ForEach-Object { Stop-Process -Id $_ -Force -ErrorAction SilentlyContinue }
	}
}

Stop-PortProcess 8080
Stop-PortProcess 8081
Stop-PortProcess 8082
Stop-PortProcess 8091

Notas:
- 8080: frontend Flask
- 8081: API runtime interna (ApiConexion)
- 8082: backend Spring
- 8091: servidor WebSocket interno del backend

Nota adicional:
- Si 8080 esta ocupado, el frontend elegira automaticamente: 8085, 8090 o 5000.


1) Levantar backend (Java/Spring)
---------------------------------
Abrir una terminal en la raiz del proyecto:

cd .\juegos1000tres-backend
.\mvnw.cmd spring-boot:run

Comprobar que responde:
(Test-NetConnection 127.0.0.1 -Port 8082).TcpTestSucceeded
(Test-NetConnection 127.0.0.1 -Port 8091).TcpTestSucceeded

Debe devolver:
True
True

Criterio de exito del backend:
- Si ves en consola "Tomcat started on port 8082" y "Started Juegos1000tresBackendApplication", el backend esta OK.
- Deja esa terminal abierta mientras pruebas el frontend.


2) Levantar frontend (Flask)
----------------------------
Abrir OTRA terminal en la raiz del proyecto:

cd .\juegos1000tres-frontend

Si estas ubicado en .\juegos1000tres-backend, usa:
cd ..\juegos1000tres-frontend

Si es la primera vez:
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Si ya tienes el entorno creado:
.\.venv\Scripts\Activate.ps1

Ejecutar frontend:
python .\main.py

El frontend queda en el puerto que muestre la consola (URL: ...).
Normalmente sera 8080; si esta ocupado, usara 8085/8090/5000.


3) Probar la sala compartida (seleccion de juego + rol)
-------------------------------------------------------
Abrir en navegador usando el puerto real del frontend:
http://127.0.0.1:<PUERTO_FRONTEND>/pruebas/sala-mentira

En la pantalla de entrada:
1) Escribe nombre.
2) Elige juego (Space Invaders o Prueba WebSocket).
3) Elige tipo de cliente (Jugador o Pantalla compartida).
4) Pulsa "Entrar a la sala".

Dentro de la sala:
1) Pulsa "Iniciar juego" o "Iniciar pantalla compartida".
2) Si quieres otro cliente, abre otra pestana en la misma URL de la sala y entra con otro nombre/rol.
3) Puedes usar "Cambiar nombre o juego" para volver y elegir otro juego desde la misma sala.


4) Flujo recomendado para Space Invaders
----------------------------------------
1) Ventana A: entra como Jugador, juego = Space Invaders.
2) Ventana B: entra como Pantalla compartida, juego = Space Invaders.
3) Inicia ambos y juega.
4) Al volver a la sala del jugador, se muestra el estado/resultado de partida.


5) Flujo recomendado para Prueba WebSocket
------------------------------------------
1) Ventana A: entra como Jugador, juego = Prueba WebSocket.
2) Ventana B: entra como Pantalla compartida, juego = Prueba WebSocket.
3) En Jugador, escribe textos y pulsa Enviar.
4) Verifica:
   - En Jugador aparece el historial global (sin autor).
   - En Pantalla aparece la lista de jugadores y sus palabras.

8) Error comun y solucion rapida
--------------------------------
Si aparece "Port 8082 was already in use" o "Address already in use",
vuelve al paso 0 (limpiar puertos) y arranca de nuevo backend y frontend.

Si aparece al final "Process terminated with exit code: -1" despues de haber arrancado bien:
- Suele significar que el proceso fue detenido externamente (terminal cerrada/cancelada).
- Solucion: vuelve a lanzar .\mvnw.cmd spring-boot:run y deja la terminal abierta.

Si quieres forzar un puerto especifico para el frontend:
$env:ARCADE_HUB_PORT="8085"
python .\main.py
